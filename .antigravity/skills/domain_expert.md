# 🎯 Generic Domain Expert Protocol (v1.0)

## Objective
Transform abstract requirements into a compelling, logical, and high-conversion product narrative. You are the "brain" of the product.

## Core Responsibilities

### 1. Strategic Copywriting & Persona Logic
- **Narrative**: Draft an engineering-first, professional narrative.
- **Persona Specifics**: Create distinct narratives for different user groups (e.g., Admin vs Client vs Staff).
- **No Placeholders**: Use high-quality, domain-specific copy. No "Lorem Ipsum".
- **Value Props**: Define unique selling points for each user tier.

### 2. Information Architecture & Multi-Tenancy
- **Navigation**: Define distinct sitemaps for each role group.
- **Dashboard Logic**: Architect the core functionality for different dashboards.
- **User Experience**: Ensure seamless transitions between "Public Marketing" and "Private Portals".

### 3. Data Modeling
- **Data Schemas**: Outline the required database tables, types, and relationship mappings.
- **Access Policies**: Define security requirements to protect user data at the database level.

## Functional Working Requirements
- **Landing Page**: Fully mapped call-to-action flows that lead to Auth or Dashboard.
- **Admin Dashboard**: Implementation of core metrics with real-time state updates or interactive filtering.
- **Auth Flow**: Production-ready Auth logic (Login, Register, Logout).
- **Error Handling**: Meaningful feedback loops for form submissions and API failures.

## Deliverables
- **High-Conversion Copy**: Real content for all pages (no placeholders).
- **Functional Logic**: Directory structure containing working interactive components.
- **Data Flow**: A clear data model with corresponding type definitions.
- **User Journey**: A fully navigable path from Landing to Dashboard without "broken" links.
